# keta's blog

keta's personal publishing space for posts and ideas in English and Chinese.

The site is built with [Fumadocs](https://www.fumadocs.dev/) and Next.js and exported for static hosting on GitHub Pages.

## Features

- A calm, responsive reading experience with light and dark themes
- Article reading progress and typography styled for long-form prose, tables, diagrams, formulas, and code
- Posts authored in MDX, with frontmatter as the only metadata source
- Paired English and Chinese posts with one-click language switching
- Global bilingual search backed by Fumadocs and static Orama indexes
- A latest-post home page, chronological Posts archive, About, and FAQs
- RSS, generated Open Graph images, `llms.txt`, and one-click clean Markdown copying for every post
- Giscus comments backed by GitHub Discussions
- Keyboard shortcuts available on every page

Email subscriptions, contact forms, and `llms-full.txt` are not part of the site.

## Development

This repository uses npm. Install dependencies and start the local development server:

```bash
npm ci
npm run dev
```

Open <http://localhost:3000> in a browser.

Verify a change before handing it off:

```bash
npm run lint
npm run types:check
npm run build
```

The production build is written to `out/` and must remain compatible with static hosting.

## Keyboard shortcuts

| Shortcut | Action |
| --- | --- |
| `Command/Ctrl + K` | Open search |
| `D` | Toggle light and dark themes |
| `L` | Switch between English and Chinese |
| `H` | Return to the localized home page |
| `M` | Copy the current post as Markdown |

Shortcuts are ignored while typing in an input, textarea, select, or editable element. `M` is available on post pages.

## Production build

For a GitHub project page, set the public origin and repository base path during the build:

```bash
NEXT_PUBLIC_SITE_URL=https://keta1930.github.io/blog \
NEXT_PUBLIC_BASE_PATH=/blog \
npm run build
```

## Deployment

Pushes to `main` trigger [the GitHub Pages workflow](./.github/workflows/deploy-pages.yml). The workflow installs locked dependencies, runs linting and type checks, builds the static site, uploads `out/`, and deploys it to the `github-pages` environment.

The build reads the public site URL and base path from GitHub Pages, so project pages and configured custom domains use the correct links and assets. Set the repository's Pages source to **GitHub Actions** under **Settings → Pages**.

## Comments

Article comments use [Giscus](https://giscus.app/) and GitHub Discussions. Before deployment:

1. Create the public GitHub repository, enable Discussions, and install the Giscus app.
2. Create a discussion category for article comments.
3. Complete the Giscus configuration form to obtain the repository and category IDs.
4. Add these non-secret repository variables under **Settings → Secrets and variables → Actions → Variables**:

| Variable | Value |
| --- | --- |
| `GISCUS_REPO_ID` | Repository ID generated by Giscus |
| `GISCUS_CATEGORY` | Discussion category name |
| `GISCUS_CATEGORY_ID` | Discussion category ID generated by Giscus |

The deployment workflow supplies the repository name automatically. Comments remain hidden when the complete configuration is unavailable. For local development, copy `.env.example` to `.env.local` and fill in the four `NEXT_PUBLIC_GISCUS_*` values.

## Publishing a post

Posts are sourced only from `content/docs/`. Create one directory per article, named with its publication date and kebab-case English title:

```text
content/docs/2026-07-22-a-post-title/
├── index.mdx
├── index.zh.mdx
└── assets/
```

`index.mdx` is English, `index.zh.mdx` is Chinese, and article-specific images belong in `assets/`. Each MDX file must contain complete frontmatter with the same stable `slug`:

```yaml
---
title: A post title
slug: a-post-title
description: A short summary used by lists, search, RSS, and metadata.
publishedAt: '2026-07-22'
updatedAt: '2026-07-22'
readingTime: 5
---
```

Dates use `YYYY-MM-DD`; `readingTime` is a positive whole number of minutes. The directory name organizes source files and does not control the public URL: `slug` does. Keep it unchanged after publication. The home page, Posts pages, article routes, adjacent-post navigation, search, RSS, OG metadata, and AI-readable output update automatically from these MDX files. No TSX list or route map needs editing.

Published routes are derived from the shared slug:

- English: `/posts/my-post`
- Chinese: `/zh/posts/my-post`
